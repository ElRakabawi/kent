Introduction

This Track Collection Builder is a tool which allows multiple tracks to be grouped together into one composite track or "collection." A composite track can be thought of as a container for grouping a set of similar-type annotation tracks.

Supported Data Types

The following continuous-value graphing track types can be added to a collection:

  • Wiggle Track Format (WIG)
  • bigWig Track Format
  • bedGraph Track Format

Track Sources

To add tracks into a new collection, they must first be visible in the browser. The following track sources can be added to a single genome browser (e.g., tracks from various sources can be loaded onto the human GRCh38/hg38 genome browser, but not both the GRCh37/hg19 and the hg38 browser):

  • Native tracks (tracks natively hosted by the UCSC Genome Browser)
  • Custom tracks(user-generated)
  • Hub tracks from your own unlisted hub or from a public hub.
